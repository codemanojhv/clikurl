# Decisions

- **Lazy Database Initialization**: The database client in `src/lib/db.ts` is initialized lazily inside `getDb()`. This prevents compilation errors if the local `.env.local` file contains connection placeholders during build-time page evaluation.
- **Manual Bypass Override**: Added an optional `?bypass=true` parameter to the verification endpoint so developers/users can bypass live CNAME checks for local/mock validation in the dashboard.
- **Parameter Aliases**: Enabled fallback mappings for developer inputs (`destination` -> `url`, `slug` -> `customAlias`, `domain` -> `customDomain`) so developers can utilize standard integrations.
